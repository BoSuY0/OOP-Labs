### 1.3 Секції класу

```cpp
class Student
{
private:
    // ── Приватні поля (інкапсуляція) ──
    std::string firstName;   // ім'я студента
    std::string lastName;    // прізвище студента
    int         age;         // вік студента (1–150)

    // ── Статичний лічильник створених об'єктів ──
    static int objectCount;

    // ── Приватний допоміжний метод — валідація віку ──
    static bool isValidAge(int value);

public:
    // Конструктори, деструктор, гетери, сетери,
    // методи відображення, статичні методи
};
```

### 1.4 Конструктори

#### 1.4.1 Конструктор за замовчуванням

```cpp
Student()
    : firstName("Невідомий"),
      lastName("Невідомий"),
      age(0)
{
    ++objectCount;
    std::cout << "[Конструктор за замовчуванням] "
              << "Створено студента #" << objectCount
              << " (значення за замовчуванням).\n";
}
```

#### 1.4.2 Конструктор з параметрами (повна ініціалізація)

```cpp
Student(const std::string& first, const std::string& last, int ageVal)
    : firstName(first), lastName(last), age(ageVal)
{
    // Валідація імені
    if (!isValidName(first)) {
        throw std::invalid_argument("Некоректне ім'я: ...");
    }
    // Валідація прізвища
    if (!isValidName(last)) {
        throw std::invalid_argument("Некоректне прізвище: ...");
    }
    // Валідація віку
    if (!isValidAge(ageVal)) {
        throw std::invalid_argument("Некоректний вік: ...");
    }
    ++objectCount;
}
```

#### 1.4.3 Делегуючий конструктор (2 параметри)

```cpp
Student(const std::string& first, const std::string& last)
    : Student(first, last, 18)  // делегування до конструктора з 3 параметрами
{
    std::cout << "  (Вік встановлено за замовчуванням: 18)\n";
}
```

#### 1.4.4 Конструктор копіювання

```cpp
Student(const Student& other)
    : firstName(other.firstName),
      lastName(other.lastName),
      age(other.age)
{
    ++objectCount;
}
```

### 1.5 Деструктор

```cpp
~Student()
{
    --objectCount;
    std::cout << "[Деструктор] Об'єкт студента \""
              << firstName << " " << lastName
              << "\" знищено. Залишилось об'єктів: "
              << objectCount << ".\n";
}
```

### 1.6 Гетери та сетери

```cpp
// ── Гетери ──
const std::string& getFirstName() const { return firstName; }
const std::string& getLastName()  const { return lastName; }
std::string getFullName()         const { return firstName + " " + lastName; }
int getAge()                      const { return age; }

// ── Сетери (з валідацією) ──
void setFirstName(const std::string& newFirstName) {
    if (!isValidName(newFirstName))
        throw std::invalid_argument("Некоректне ім'я: ...");
    firstName = newFirstName;
}

void setLastName(const std::string& newLastName) {
    if (!isValidName(newLastName))
        throw std::invalid_argument("Некоректне прізвище: ...");
    lastName = newLastName;
}

void setAge(int newAge) {
    if (!isValidAge(newAge))
        throw std::invalid_argument("Некоректний вік: ...");
    age = newAge;
}
```

## 4. Відповіді на контрольні питання

### 4.1 Що таке інкапсуляція?

**Інкапсуляція** (від лат. *in capsula* — «у капсулі») — один із трьох фундаментальних принципів ООП, що полягає в:

1. **Об'єднанні** даних (полів) і методів, що працюють з цими даними, в одному класі.
2. **Приховуванні** внутрішнього стану об'єкта від зовнішнього коду.
3. **Контрольованому доступі** до внутрішніх даних через публічний інтерфейс (гетери / сетери).

#### Навіщо потрібна інкапсуляція?

| Переваги | Пояснення |
|----------|-----------|
| **Захист від некоректних даних** | Сетер `setAge()` перевіряє, що вік у діапазоні 1–150, не даючи записати `-5` або `200` |
| **Контроль доступу** | Можна дозволити лише читання (`getAge()`), заборонивши зовнішню модифікацію поля |
| **Гнучкість реалізації** | Внутрішнє представлення можна змінити (наприклад, зберігати дату народження замість віку), не змінюючи інтерфейс |
| **Спрощення налагодження** | Усі зміни проходять через відомі точки (сетери), які можна логувати |
| **Зменшення зв'язності** | Зовнішній код залежить лише від інтерфейсу, а не від реалізації |

#### Приклад з лабораторної роботи

```cpp
// Поля — приватні (private) → зовнішній код НЕ може їх змінити
private:
    std::string firstName;
    int age;

// Доступ — лише через публічні методи
public:
    void setAge(int newAge) {
        if (newAge < 1 || newAge > 150)
            throw std::invalid_argument("Некоректний вік");
        age = newAge;  // зміна лише після перевірки
    }
```

### 4.2 Які бувають модифікатори доступу в C++?

У C++ є **три модифікатори (специфікатори) доступу**:

| Специфікатор | В самому класі | У дочірньому класі | Зовні класу | Типове використання |
|---|---|---|---|---|
| `public` | ✅ Так | ✅ Так | ✅ Так | Методи-інтерфейс (гетери, сетери, `displayInfo()`) |
| `protected` | ✅ Так | ✅ Так | ❌ Ні | Поля/методи для успадкування |
| `private` | ✅ Так | ❌ Ні | ❌ Ні | Внутрішні поля (`firstName`, `age`) та допоміжні методи |

#### Відмінність `class` від `struct`

| Ознака | `class` | `struct` |
|--------|---------|----------|
| Доступ за замовчуванням | `private` | `public` |
| Успадкування за замовчуванням | `private` | `public` |
| Семантика | Повноцінний об'єкт з інкапсуляцією | Простий агрегат даних |

**Приклад з лабораторної:**

```cpp
class Student
{
private:    // ← недоступно зовні
    std::string firstName;
    int age;
    static bool isValidAge(int value);

public:    // ← доступно всім
    Student(const std::string& first, const std::string& last, int ageVal);
    const std::string& getFirstName() const;
    void setAge(int newAge);
    void displayInfo() const;
    static int getObjectCount();
};
```

## 5. Відповіді на додаткові питання для самоперевірки

### 5.1 Як у C++ оголошується клас і чим він відрізняється від структури (`struct`)?

**Клас** оголошується ключовим словом `class`, після чого в фігурних дужках розміщують секції `private`, `protected`, `public` з полями та методами. Оголошення завершується крапкою з комою.

```cpp
class MyClass {
private:
    int data;
public:
    MyClass(int d) : data(d) {}
    int getData() const { return data; }
};
```

**Різниця від `struct`:** в `class` доступ за замовчуванням — `private`, в `struct` — `public`. В усьому іншому вони функціонально ідентичні в C++.

---

### 5.2 Які є секції доступу в класі і коли доцільно використовувати кожну з них?

- **`private`** — для полів-даних і внутрішніх допоміжних методів. Це основний засіб інкапсуляції.
- **`public`** — для конструкторів, деструктора, гетерів/сетерів та будь-яких методів, що формують «інтерфейс» класу.
- **`protected`** — для полів і методів, які мають бути доступні в дочірніх класах при успадкуванні, але не ззовні.

---

### 5.3 Для чого потрібні конструктори та деструктори? Як правильно їх оголошувати?

**Конструктор:**
- Викликається автоматично при створенні об'єкта.
- Ім'я збігається з іменем класу, не має типу повернення.
- Може бути перевантажений (декілька варіантів з різними параметрами).
- Може делегувати іншому конструктору (C++11).

```cpp
Student();                                              // за замовчуванням
Student(const std::string& f, const std::string& l, int a); // з параметрами
Student(const Student& other);                          // копіювання
```

**Деструктор:**
- Викликається автоматично при знищенні об'єкта (вихід зі скоупу, `delete`).
- Ім'я — `~ClassName()`, без параметрів, без типу повернення.
- Є лише один (не перевантажується).
- Використовується для звільнення ресурсів.

```cpp
~Student() {
    --objectCount;
    std::cout << "Об'єкт знищено\n";
}
```

---

### 5.4 Як у C++ створюються об'єкти класу та викликаються їхні методи?

```cpp
// Створення на стеку
Student s1;                          // конструктор за замовчуванням
Student s2("Олена", "Шевченко", 20); // конструктор з параметрами
Student s3(s2);                      // конструктор копіювання

// Виклик методів — через оператор "крапка" (.)
s2.displayInfo();
int age = s2.getAge();
s1.setFirstName("Марія");

// Створення в динамічній пам'яті
Student* ptr = new Student("Іван", "Франко", 25);
ptr->displayInfo();  // виклик через оператор "->"
delete ptr;          // явне знищення → деструктор
```

### 5.6 Для чого потрібні методи-гетери (`get`) та методи-сетери (`set`)?

- **Гетер** — повертає значення приватного поля, не змінюючи об'єкт (`const`-метод).
- **Сетер** — змінює значення приватного поля, попередньо **валідуючи** нове значення.

**Переваги:**
1. Дозволяють контролювати, які дані записуються в об'єкт.
2. Можна зробити поле «лише для читання» (тільки гетер, без сетера).
3. Дозволяють додати логування, кешування або інші побічні ефекти.
4. Зовнішній код не знає про внутрішню структуру класу.

---

### 5.7 Що таке статичні (`static`) члени класу і як їх використовувати?

**Статичне поле** (`static int objectCount`) — існує в єдиному екземплярі для **всіх** об'єктів класу. Не прив'язане до конкретного об'єкта.

**Статичний метод** (`static int getObjectCount()`) — може бути викликаний без створення об'єкта, через ім'я класу: `Student::getObjectCount()`. Не має доступу до `this` та нестатичних полів.

```cpp
// Оголошення в класі
static int objectCount;
static int getObjectCount() { return objectCount; }

// Ініціалізація за межами класу (один раз)
int Student::objectCount = 0;

// Виклик
std::cout << Student::getObjectCount();  // без об'єкта
```

**У лабораторній** статичний лічильник `objectCount` відстежує кількість «живих» об'єктів: інкрементується в кожному конструкторі, декрементується в деструкторі.

---

### 5.8 Як забезпечити, щоб при невірному виклику методу стан об'єкта лишався коректним?

1. **Валідація у сетерах** — перевірка вхідних даних перед присвоєнням.
2. **Кидання виключень** (`throw std::invalid_argument(...)`) — якщо дані некоректні, виключення кидається **до** зміни стану об'єкта, тому об'єкт залишається в попередньому валідному стані.
3. **Конструктор з валідацією** — об'єкт або створюється коректно, або не створюється взагалі (strong exception guarantee).

**Демонстрація у програмі (Розділ 3):**

```cpp
try {
    s2.setLastName("123Invalid!");  // некоректне прізвище
} catch (const std::invalid_argument& e) {
    std::cout << e.what();
}
// s2 не змінився — об'єкт залишається в коректному стані
s2.displayInfo();  // досі "Олена Шевченко"
```
