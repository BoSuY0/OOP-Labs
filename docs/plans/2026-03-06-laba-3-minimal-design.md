# Laba 3 Minimal Design

**Дата:** 2026-03-06

**Мета:** спростити лабораторну роботу №3 до мінімально потрібної реалізації без втрати вимог завдання.

## Що залишаємо

- Базовий клас `Person` з полями `name` і `age`
- Похідний клас `Student : public Person` з полем `specialty`
- Конструктори базового і похідного класів
- Віртуальний метод для показу повної інформації
- Віртуальний деструктор у `Person`
- Демонстрацію поліморфізму через покажчик `Person*`

## Що прибираємо

- `Teacher`
- окремі інтерфейси для множинного наслідування
- діамантове наслідування
- зайву валідацію, лічильники, копіювання, сетери/гетери
- складне форматування виводу

## Структура програми

- `Person` зберігає ім'я та вік і має `virtual void show() const`
- `Student` викликає конструктор `Person(name, age)` у списку ініціалізації
- `Student::show()` доповнює інформацію спеціальністю
- `main()` створює один `Person`, один `Student`, а далі викликає `show()` через `Person*`

## Очікуваний результат

Код стає коротким і читабельним, але все ще демонструє:

- наслідування `public`
- розширення базового класу
- перевизначення методу
- поліморфний виклик через покажчик на базовий клас
